2. Form & Integration Requirements

Before forms are built on the React site, we need to align on what each form must do. The recent incident showed what happens when integrations are assumed rather than specified.

Every lead-capturing form on the new site must:

  1. Map fields correctly to HubSpot — firstname, lastname, email, phone
  2. Connect the Zapier webhook (9wymd7m4@robot.zapier.com)
  3. Pass the source page URL so leads can be attributed
  4. Pass transaction_type (purchase / refinance / commercial / build)
  5. Pass quiz answer data where applicable (not hardcoded values)
  6. Show a success message only after the backend confirms receipt

3. Lead Flow: Website → Monday.com

The current lead flow is working again after the April fix:

Website form → CF7 → HubSpot (Collected Forms) → Zapier → Monday.com → Broker notification

The concern is ensuring this flow doesn't regress — and that the React migration doesn't break any part of the chain.

Questions:

  • On the React site, will forms still go through CF7, or is there a new submission mechanism?
  • If the submission path changes, how do we ensure HubSpot, Zapier, and Monday.com integrations are preserved?
  • What monitoring or alerting can we put in place so that if any step in the chain breaks, we know immediately — not months later?

4. QA & Deployment Process

The React migration is a full rebuild — we need to understand how CRS plans to test it before it goes live.

What we'd expect from CRS as the developers:

  • Test cases for form submissions — verifying data arrives in HubSpot with correct field mappings
  • Integration verification — confirming webhooks (Zapier), CRM field mapping, and lead routing work end-to-end
  • A staging environment where we can review and verify before production deployment
  • A sign-off process — who confirms the build is ready for production?

Questions for CRS:

  • What is your QA process for the React migration?
  • Will you be providing test cases or a test report before go-live?
  • How do you plan to verify that every form on the new site delivers data correctly to HubSpot and Zapier?

What we've built as a safety net:

  • A 142-test automated suite (Playwright) covering all forms, quizzes, and calculators — available as an additional verification layer (test coverage details)
  • HubSpot active list alerts that flag corrupted data in real time (being set up this week)
