Architecture: Clever Lead Capture Funnel

The data flow is: Quiz inputs → Client-side calculation → DOM update (report sections) → Email gate (CF7 #67146) → User sees report → Assessment CTA (CF7 #67147).

The quiz data (property type, price, deposit, income, etc.) is used only for the client-side report generation — it is not submitted to HubSpot. Only the contact details (name, email, phone) from forms #67146 and #67147 reach HubSpot.

This is the same pattern as the homepage purchase path where situation/finance fields feed the borrowing calculator but aren't submitted to the backend.

Status: Working as Designed

Both CF7 forms (#67146 and #67147) are HubSpot-bound and capture contact details. The quiz answers intentionally stay client-side to power the report generator. The page also loads the standard popup forms (11832 ×4, 14462 ×3, 14422, 14296, 14061, 14060, 14059) via the global template.

C-02: Equity Calculator — Broken Hero Banner

C-02 · CALCULATOR

Equity Calculator — Two Calculators, One Broken

/equity-calculator/ Custom JS Display only

There are two separate calculators on this page. The one further down the page works. The hero banner calculator at the top is completely broken.

Working Calculator (below the fold)

Located inside #h_equity_calculator. Has proper inputs (equity_loan_amount, equity_property_value, investment/owner-occupier toggle) and a "Calculate" button with onclick="h_equity_calculator()" that runs the actual equity math. This one works fine.

Broken Hero Banner Calculator (top of page)

Three problems:

1. Orphaned input. The input is <input class="input-banner" placeholder="e.g. 200,000"> with no name, no id, and zero JavaScript anywhere on the page (160k chars of scripts checked) that references it. Nothing reads its value. It's a completely dead input field.

2. "Calculate" button is a self-link. It's <a href="/equity-calculator" class="btn_yellow">Calculate</a> — which is the current page. Clicking it reloads the page and the input value is lost. No onclick handler, no JS attached.

3. Missing input. There's only one input field, but an equity calculation needs two values (property value AND loan balance). Even if the JS was wired up, you can't calculate equity from one number.

This looks like someone added a hero banner template with a decorative calculator input (probably copied from another page's design) but never connected it to anything.

Recommended Fix

Option A: Wire the banner input to scroll down to the working calculator and pre-fill one of its fields.

Option B: Remove the fake banner calculator entirely since it does nothing but confuse users.

Option B is simpler and avoids maintaining two calculator UIs.
