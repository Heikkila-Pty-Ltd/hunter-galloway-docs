1. The Problem

Quiz answers are never submitted

The /lower-your-rate/ page presents a 5-step refinance quiz. The user selects their interest rate, loan amount, loan term, and refinancing priority before entering contact details.

However, the underlying CF7 form (ID 14462) only contains 3 fields: name, email, and phone. None of the quiz selections are wired into the form.

The attribute data-hs-cf-bound="true" confirms HubSpot integration is active — submissions reach HubSpot but arrive with no quiz data whatsoever.

Evidence in HubSpot

12 submissions from /lower-your-rate/ appear in HubSpot with name, email, and phone populated but no transaction type and no quiz answer data. This is consistent with the form only capturing contact fields.

5. The Fix

Step A: Add hidden fields to the CF7 form

In the CF7 form editor for form ID 14462, add these hidden fields before the submit button:

[hidden interestRate default:get]
[hidden loanAmount default:get]
[hidden loanTerm default:get]
[hidden refiPriority default:get]
[hidden transaction_type default:refinance]

Step B: Populate hidden fields via JavaScript

Add this script to the /lower-your-rate/ page (or in the theme's custom JS). It should run after the quiz UI sets the selections and before CF7 submits.

document.addEventListener('DOMContentLoaded', function () {
  var form = document.querySelector('.wpcf7-form');
  if (!form) return;

  // Map quiz step data attributes/variables to hidden field names
  var fieldMap = {
    interestRate:  '[name="interestRate"]',
    loanAmount:    '[name="loanAmount"]',
    loanTerm:      '[name="loanTerm"]',
    refiPriority:  '[name="refiPriority"]'
  };

  form.addEventListener('submit', function () {
    // quizData should be populated by the existing quiz step JS
    // Adjust property names to match the quiz's actual variable/store
    var quizData = window.quizData || {};

    Object.keys(fieldMap).forEach(function (key) {
      var input = form.querySelector(fieldMap[key]);
      if (input && quizData[key]) {
        input.value = quizData[key];
      }
    });
  });
});

Note on quiz JS integration

The snippet above assumes quiz answers are stored on window.quizData. Inspect the existing quiz JavaScript on the live page to find the actual variable or data store that holds selections from steps 1-4, and update the property names accordingly.

Step C: HubSpot field mapping

After adding the hidden fields to CF7, update the HubSpot CF7 integration mapping to include:

CF7 Field HubSpot Property Property Type
interestRate interest_rate Single-line text (or dropdown)
loanAmount loan_amount Single-line text (or dropdown)
loanTerm loan_term Single-line text (or dropdown)
refiPriority refi_priority Single-line text (or dropdown)
transaction_type transaction_type Single-line text

Expected outcome

After applying all three steps, every submission from /lower-your-rate/ will include the user's quiz selections alongside their contact details. HubSpot contacts will show the full refinancing context, and transaction_type will be set to refinance automatically.
